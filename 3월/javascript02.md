# JavaScript

## 자바스크립트란?
### 자바스크립트 특징
- 프로그래밍은 언어, 의사소통이 목적(머신)
- 전기 신호의 논리 값을 전달하는 어려움
- 서로에 대한 언어를 모르면 통역사가 필요함(컴파일러)
- 과거엔 0101 비트어를 전송했음
#### 저급언어, 중간언어
- 저급언어 기계어
- 중간언어인 바이트코드
- 사람 언어에 가까운 자바, 자바스크립트 90년대에 등장 `고수준 언어`
- 인기에 편승할려고 자바 스크립트로 명명
- 스크립트 언어의 특징은 컴파일이 필요없음
#### 컴파일 언어와 스크립트랑 차이점
- 컴파일 언어는 번역 (번역본을 써서 전달, 여러번 사용 가능, 느림)
- 스크립트는 통역 (통역은 바로바로 전달, 기록이 안됨, 빠름)
- 예전엔 하드웨어가 좋지 않아서 인기가 없었음
#### 노드 JS
- 노드 JS 환경으로 지도웹을 만듬, 서버 프로그래밍도 가능
- 2016년 기준으로 프로그래밍 언어 됨
- 서버에서 실행되는 자바스크립트 환경을 node.js
- `자바스크립트 엔진`, 모든 디바이스에 동작
- 브라우저는 기본적으로 내장되어있음
### 브라우저에서 할 수 있는 일
- 메모리나 CPU 저수준 조작을 허용하지 않음
- 웹페이지 조작, 클라이언트 서버의 상호작용
  - 페이지에 새로운 HTML 추가, 스타일 수정
  - 마우스 클릭 등과 같은 이벤트 처리
  - 네트워크 통해 원격 서버 요청과 파일 다운로드 및 업로드(AJAX)
  - 쿠키를 가져오거나 설정하기
  - 클라이언트 측에 데이터 저장하기(로컬 스토리지)
### 브라우저에서 할 수 있는 일
- 로컬 데이터를 손상하는 것을 막기 위해 만들어짐
  - 파일 입출력
  - Same Origin Policy
### 자바스크립트 관련 언어들
- 자바스크립트는 짬뽕 언어(다 가지고 옴)
- 2015년 대규모 개정 MDN, ES6
- Class문법, 화살표 함수(람다)
- 매년 혁신을 계속 냄, 객체를 생성하는 방법이 7개 넘음
- 혼란이 가중 됨, 획일된 표준이 다름
- 코딩 짜는 방식이 다르지만 동작은 같음

## 자바스크립트 기본
### 변수와 상수
- var, let, const 키워드
- 메모리 공간에 이름을 달아줌
- `username = 'abc'` assignment, 우항에 있는 값을 좌항에 저장하세요
- 자바스크립트의 문제점, 안되면 안되는 것을 표시 안해줌
- `undifiend` 나오면 값이 없다는 것을 알아야 함
- 자바 스크립트는 그릇이 변함 (메모리 부담 증가)
### 변수의 재할당
- 변동가능한 데이터
- 상수 : 항상 똑같은 값, 한번 할당된 값을 변경할 수 없음
### 변수 이름 규칙
- `var my_home_address` 스네이크 케이스는 자바, 자바 스크립트에 쓰지 말기
- `var myHomeAddress` 케멀 케이스를 자바, 자바 스크립트에 쓰기
- 특수문자는 `$_`  파이프 표기법
- `-`은 안됨, 케밥 케이스는 HTML, CSS에 씀
- `var UserPassword` 파스칼 케이스, 클래스랑 생성자 이름에 사용
- `var $button` 헝가리안 케이스, `$`는 태그를 의미 type+Hungrian case
### 상수(const)
- `const`사용

### 바람직한 변수명
- 변수명은 간결하고 명확해야 함
- 구글 홈페이지 html의 id는 암호화되어 있음
